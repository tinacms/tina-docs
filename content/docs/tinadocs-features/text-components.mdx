## Preloaded Images

Preloaded images are optimized image components that prevent layout shift by storing dimensions with the image. This improves Core Web Vitals and provides a better user experience.

### Why Use Preloaded Images?

Regular markdown images (`![alt](path)`) load dimensions at runtime, which can cause:
- Layout shift as the page loads (poor CLS score)
- Slower initial render
- Flickering content

Preloaded images solve this by:
- âœ… Storing width and height with the image
- âœ… Preventing layout shift (better CLS)
- âœ… Showing a loading skeleton while loading
- âœ… Smooth fade-in transition when loaded
- âœ… Built-in lightbox support

### Loading States

Both image types show a loading skeleton while the image loads:
- Regular images: Load skeleton + fetch dimensions
- Preloaded images: Load skeleton only (dimensions already known)

### When to Use Each Type

**Use regular markdown images** when:
- Quick documentation or notes
- Image optimization is not critical
- You don't mind runtime dimension loading

**Use preloaded images** when:
- Page performance matters (Core Web Vitals)
- Professional documentation
- Preventing layout shift is important
- You want consistent loading experience
