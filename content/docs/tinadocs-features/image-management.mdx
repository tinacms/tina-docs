TinaDocs provides optimized image handling that prevents layout shift and integrates seamlessly with TinaCMS.

## Image Storage with TinaCloud

TinaDocs uses a hybrid approach for image storage:

**Repo-Based Storage:**

* All images are stored in your Git repository (`/public/img/`)
* Version controlled with your content
* No external dependencies

**TinaCloud Image Serving:**

* **During Editing (TinaCMS Admin):** Images served from TinaCloud with bespoke editing capabilities
* **On Static Pages (Production):** Images served from your project with Next.js optimization

This gives you powerful editing tools in the CMS and optimal performance in production.

## Image Components

### Regular Markdown Images

Standard markdown syntax loads dimensions at runtime:

```markdown
![Alt text](/img/example.png "Optional caption")
```

**Use for:** Quick documentation, drafts, internal notes

### Preloaded Images

Optimized components with stored dimensions prevent layout shift:

```jsx
<preloadedImage
  image={{
    src: "/img/example.png",
    width: 1920,
    height: 1080,
    alt: "Description"
  }}
  caption="Optional caption"
/>
```

**Use for:** Production documentation, public-facing content, performance-critical pages

## Migration Scripts

### Migrate Images

Automatically converts images to the optimized format:

```bash
pnpm run migrate-images
```

**What it does:**

* Scans all MDX files
* Finds markdown images: `![alt](path "caption")`
* Finds images in accordion and scrollShowcase components
* Loads each image to get dimensions
* Converts to preloaded format
* Creates `.backup` files

**Example transformation:**

```markdown
![Screenshot](/img/example.png "Example")
```

Becomes:

```jsx
<preloadedImage
  image={{
    src: "/img/example.png",
    width: 1920,
    height: 1080,
    alt: "Screenshot"
  }}
  caption="Example"
/>
```

### Cleanup Backups

Remove backup files after verifying migration:

```bash
pnpm run cleanup-backups
```

Lists all `.backup` files and asks for confirmation before deletion.

## Workflow

1. **Migrate:** `pnpm run migrate-images`
2. **Review:** Check images display correctly
3. **Cleanup:** `pnpm run cleanup-backups`
4. **Commit:** Push changes to version control

## Custom TinaCMS Field

The custom image field automatically captures dimensions when you upload or select images in TinaCMS. Width and height are stored automatically - you just need to add alt text.

## Performance Benefits

**Core Web Vitals:**

* Preloaded images: CLS score of 0.0 (no layout shift)
* Regular images: CLS score of 0.1-0.25 (noticeable shift)

**User Experience:**

* No content jumping during load
* Smooth loading animations
* Professional appearance
* Better accessibility
